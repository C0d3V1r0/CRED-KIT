# Release Checklist

## 1) Перед прогоном
- Обновить рабочую ветку и убедиться, что нет незавершённых конфликтов.
- Проверить конфиг в `config/appConfig.ts` (версия, telemetry/web-vitals endpoint, донат/bug-report URL).

## 2) Обязательные команды
- `npm run test:clean`
- `npm run test:quick`
- `npm run test:e2e`
- `npm run test:lighthouse`
- `npm run test:budget`

Если любой шаг падает, релиз блокируется до исправления.

## 3) Обязательная ручная проверка
- Локализация: переключение RU/EN и сохранение выбора после перезагрузки.
- Character: изменение статов/денег/имени, затем перезагрузка и проверка автосохранения.
- Import/Export: экспорт полного payload, повторный импорт и валидация корректного восстановления.
- Репорт в «О проекте»: открыть модалку по кнопке «Репорт», проверить валидацию формы, успешный submit и сброс после успеха.
- Mobile smoke: отсутствие page-level горизонтального скролла на основных вкладках.
- Offline banner: корректное отображение индикатора offline/online.

## 4) Артефакты и отчёты
- Сохранить итог `coverage/` и `test-results/` (если используются в CI/отчётах релиза).
- Зафиксировать результат Lighthouse и budget-проверки в описании релиза.
